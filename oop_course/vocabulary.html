📚 Key terms

These are the key OOP terms for the Cambridge 9618 course. Learn the wording, but also practise recognising each idea in code.

TermMeaning
AbstractionThe process of focusing on essential features of an object and ignoring non-essential details.
AggregationA has-a relationship where one class contains a reference to another class, but each class can exist independently.
ClassA blueprint or template for creating objects that define the properties and behaviours of those objects.
ConstructorA special method that tells Python how to create an object of this class. In Python this is always called __init__.
ContainmentThe process by which one class can contain other classes or objects. It is often linked to aggregation.
GetterA method whose purpose is to get a value from within an object.
InheritWhen a class is able to use all of the attributes and methods from another class.
InheritanceThe process in which the methods and data from a superclass or base class are copied to a derived class.
InstanceA specific example of an object created from a class.
MethodA function called on an object, allowing interaction with that object.
ObjectAn instance of a class.
Object oriented programmingA programming methodology that uses self-contained objects, which contain methods and data, and communicate with each other.
PolymorphismThe ability of an object to take many forms. In OOP this often means a child class method overrides a parent class method with the same name.
PropertyData or methods within an object that perform a named action.
SetterA method whose purpose is to set a value within an object.
AttributeA variable stored inside an object, often written using self in Python.
Private attributeAn attribute intended to be hidden from direct access outside the class, commonly shown with a double underscore in Python.
SuperclassThe parent or base class that child classes inherit from.
SubclassThe child or derived class that inherits from a parent class.
OverrideWhen a child class provides its own version of a method from the parent class.
selfThe name commonly used in Python to refer to the current object inside a class.
