1. The Core Pseudocode Command Set (Both Syllabuses)

For both CAIE IGCSE (0478) and A-Level (9618), Cambridge expects you to use the standard capitalized pseudocode commands to manage files.

Operation Syntax What it actually does
Open (Read) OPENFILE "data.txt" FOR READ Prepares an existing file to load data into the program.
Open (Write) OPENFILE "data.txt" FOR WRITE Wipes the file clean (or creates it) to start saving fresh data.
Open (Append) OPENFILE "data.txt" FOR APPEND Prepares the file to add data onto the very end without deleting anything.
Close CLOSEFILE "data.txt" Finalizes changes and safely saves them to secondary storage.
2. IGCSE (0478) Requirements: Line by Line

At IGCSE level, file handling is kept simple. You are treated as if you are interacting with individual, sequenced data items or a fixed number of rows.

The Key Commands

  • Reading: READFILE "data.txt", MyVariable
  • Writing: WRITEFILE "data.txt", MyVariable

Distinct IGCSE Characteristics

  • Fixed Loops or Sequential Steps: IGCSE questions heavily rely on you knowing exactly how many lines are in the file or just reading a few specific lines sequentially.
  • No EOF Function Required: The official 0478 pseudocode guide does not explicitly mandate the use of an End-of-File check (EOF()). You are usually reading a fixed count of items (e.g., using a FOR loop).

IGCSE Writing Example:

// Saving a user's name and high score
DECLARE Name : STRING
DECLARE Score : INTEGER
OUTPUT "Enter name and score: "
INPUT Name, Score

OPENFILE "scores.txt" FOR WRITE
WRITEFILE "scores.txt", Name
WRITEFILE "scores.txt", Score
CLOSEFILE "scores.txt"
3. A-Level (9618) Requirements: Dynamic & Complex

Once you step up to 9618, files are no longer just simple storage boxes; they require active management.

The Key Additions

  • EOF(<filename>) Function: This is mandatory knowledge for 9618. You must use it inside a conditional loop (typically WHILE NOT EOF("filename")) to process text files where the length or number of lines is unknown.
  • File Records & Data Structures: You are often expected to read data out of a file and instantly map it into an Array of Records or user-defined data types.
  • Sequential vs. Random Files (Paper 3 / A2): If you are doing the full A-Level, you must also master Random Files (OPENFILE "data.txt" FOR RANDOM), which use records, field lengths, and the SEEK command to edit specific lines without reading the entire file from the top.

A-Level Reading/Editing Example (Sequential Text File):

// Reading an unknown amount of text and looking for a specific value to edit/count
DECLARE LineText : STRING

OPENFILE "database.txt" FOR READ

WHILE NOT EOF("database.txt") DO
    READFILE "database.txt", LineText
    IF LineText = "TargetValue" THEN
        OUTPUT "Found target record."
    ENDIF
ENDWHILE

CLOSEFILE "database.txt"
Summary of the Core Differences
Feature IGCSE (047CAIE 0478) A-Level (CAIE 9618)
File Types Simple text files only. Text files, CSV files, and Random/Binary files.
Tracking File End Fixed FOR loops or static linear execution. Must use the WHILE NOT EOF("filename") loop construct.
Editing Data Rewrite the whole file from scratch using FOR WRITE. Read into memory structures (Arrays/Records), alter data, and write back out (or use Random File SEEK).
Data Mapping Variables or 1D arrays. Composite Data Types (Records), 2D Arrays, or Object-Orientated classes (Paper 4).
