Recursion vocabulary

Key terms
Core terms
  • Recursion: a function or procedure solving a problem by calling itself.
  • Base case: the stopping condition that gives a direct answer.
  • General case: the part that reduces the problem and makes a recursive call.
  • Recursive call: the call where the subroutine calls itself.
  • Trace: following a recursive algorithm step by step.
  • Call stack: the stack used to store active subroutine calls.
  • Stack frame: saved data for one call, such as return address, parameters and local variables.
  • Unwinding: returning from calls after the base case has been reached.
  • Stack overflow: an error caused when too many calls use up stack space.
  • Iteration: repetition using loops rather than self-calling functions.
  • Memoisation: saving previous answers to avoid repeated recursive work.
Exam phrases

Good exam answers often mention: base case, general case, smaller problem, stack, return address, local variables, stack frame, unwinding and stack overflow.
