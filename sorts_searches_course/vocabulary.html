VOCABULARY: 9618 ALGORITHMS

Sorts, Searches and Big O

Reference
📘 Interactive glossary

Use this page from any lesson. The key exam habit is to connect each term to the algorithm step that causes it.

Algorithm

A precise set of steps used to solve a problem.

Linear search

Checks items one by one until the target is found or the list ends.

Binary search

Searches a sorted list by repeatedly checking the middle item and discarding half of the search space.

Bubble sort

Repeatedly compares neighbouring items and swaps them when they are in the wrong order.

Insertion sort

Builds a sorted section by inserting each new item into the correct place.

Big O notation

A way to describe how an algorithm grows as input size increases.

Worst case

The maximum amount of work an algorithm may need for an input of size n.

Best case

The minimum amount of work, often when the target is found immediately or data is already sorted.

Average case

A typical amount of work across many possible inputs.

O(1)

Constant time. The work does not grow with n.

O(log n)

Logarithmic time. The work grows slowly because the problem size is repeatedly divided.

O(n)

Linear time. The work grows in proportion to n.

O(n^2)

Quadratic time. The work grows roughly with n multiplied by n.

Comparison

Checking two values to decide whether they match or which is larger.

Swap

Exchanging the positions of two items.

Pass

One complete movement through the list during a sorting algorithm.

Middle index

The index checked by binary search, usually (low + high) // 2.

Stable explanation

An exam explanation that supports the Big O answer with a reason based on algorithm steps.
