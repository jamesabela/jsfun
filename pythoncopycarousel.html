Python Code Lab 3.1 lets you load, edit, run, display, and trace Python code in the browser. It is designed for teachers who want to demonstrate algorithms clearly and for students who want to explore code step by step.

Getting started

When the page opens, you can either:

  • Paste a raw Python file URL into the box and click Load Code.
  • Click Upload .py to load a local Python file from your computer. This also allows .txt files in case of firewall restrictions.
  • Choose one of the ready-made iGCSE algorithm examples.
  • Click Blank File to start writing your own Python code.

The ready-made examples include common algorithms such as average, counting, totalling, linear search, bubble sort, if/else, for loops, while loops, and repeat-style loops.

Loading code from a URL

To load code from another source:

  1. Copy the raw URL of a Python file.
  2. Paste it into the input box.
  3. Click Load Code.

The page URL will update automatically, so you can share the loaded version with students.

Example format:
https://jamesabela.github.io/jsfun/pythoncopy?url=RAW_FILE_URL_HERE

How to make a Gist on GitHub

If you don't know how to create a Gist, follow these steps to host your Python code so it can be loaded into Python Code Lab:

  1. Go to gist.github.com (you will need a GitHub account).
  2. Create a new Gist: Give your file a name ending in .py and type or paste your code.
  3. Click the green Create secret gist (or public gist) button.
  4. Get the Raw URL: Click the Raw button in the top right corner of your code block.
  5. Copy the URL from your browser's address bar. It will start with https://gist.githubusercontent.com/...

Paste this URL into the input box to load your code!

Edit Mode

Edit Mode is the normal working mode. In Edit Mode, you can:

  • Write, paste, or load Python code in the text editor.
  • Click Run to run the code.
  • See the output and interact with input() prompts in the console window on the right.
  • Press Reset to clear the output or cancel execution at any time.

Display Mode

Display Mode lets you visually trace your Python code line by line. When you switch to Display Mode, the editor becomes read-only, and the program is analysed to prepare for tracing.

Display Mode includes: line highlighting, step-by-step execution, automatic play mode, variable tracking, and call stack tracing. It is highly recommended for teachers explaining code to students.

Visual Tracing Interface

As you step through code, the current active line is highlighted. The panels on the right show:

  • Call Trace: The stack of active functions.
  • Variables: A table of all variables in scope and their current values.
  • Output: The program's standard output.

Quick Reference & Limitations

Control Purpose
Play / Pause Automatically step through the code at a regular interval
Step Execute the next line of code
Back Move back one step of execution
Reset Reset the visual tracing session to the beginning
Full Screen Shows the tool full screen for teaching

Teacher use cases

Teachers can use Python Code Lab 3.1 to: demonstrate standard algorithms, explain code line by line, show how variables change, teach loops, selection, functions, and recursion, create code-ordering puzzles, share code examples using a URL, present Python clearly on a large screen.

Student use cases

Students can use it to: experiment with Python code, run short programs in the browser, trace algorithms step by step, check how variables change, copy or download working examples, practise rearranging code into the correct order.

Important limitations

Python Code Lab 3.1 runs Python in the browser, so it is best for short teaching examples and algorithm demonstrations. Programs that use unsupported imports, external files, or complex packages should be downloaded and run in a local Python environment instead.

Turtle Graphics & Turtle Mode

Python Code Lab 3.1 supports in-browser Turtle graphics! When your code imports turtle, or when you switch to Turtle Mode, a canvas will automatically appear on the right side of the editor.

What's supported?

  • Movement: forward(), backward(), left(), right(), goto(), home(), and exact mathematical circle(radius, extent) drawings.
  • Pen Control: penup(), pendown(), pensize() / width(), pencolor(), and fillcolor().
  • Shapes & Fills: begin_fill() and end_fill() to fill shapes. Change cursor shapes using shape("turtle"), shape("circle"), shape("square"), or shape("classic")!
  • Screen Settings: bgcolor() to set the canvas background.
  • No-op standard functions: done(), mainloop(), and exitonclick() are supported as no-ops to prevent crashes when importing desktop Python scripts.

Line-by-Line Visual Tracing

Switch to Display Mode to trace your turtle scripts. The canvas will capture a state snapshot at every execution line, allowing you to click Step or Play to watch the turtle cursor move and draw segment-by-segment in perfect synchronization with your code!

Exporting Drawings

Click the Save Image button in the turtle graphics panel header to export and download your canvas drawing as a high-quality .png file.

Reference Links & Web Previews

Python Code Lab 3.1 automatically scans your Python code for web links (e.g., documentation, tutorials, YouTube videos) written in comments or strings.

Dynamic Links Listing

  • A Reference Links panel will appear in the right column listing all URLs found in the code.
  • Click the line badge next to any link (e.g., Ln 12) to automatically scroll to, highlight, and select that line in the code editor.
  • Click Open to view the link directly in a new browser tab.

Interactive Web Embedding & Previews

  • Click Preview next to any link to display it inside an interactive embed frame within Python Code Lab.
  • Click the Preview: On toggle button at the top of the Reference Links header to easily show or hide the preview window at any time.
  • YouTube Integration: Regular YouTube video links and shorts are automatically translated to embed links, allowing you to watch tutorials directly next to your workspace!
