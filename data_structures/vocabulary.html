VOCABULARY: 9618 DATA STRUCTURES

Stacks, Queues and Linked Lists

Reference
📘 Key terms

Use this page when a lesson mentions a technical term. These definitions are written for A-level Computer Science revision and use list-based Python examples rather than OOP.

Abstract Data Type (ADT)

A model of a data structure that describes the operations allowed, not the exact implementation.

Stack

A last in, first out structure. The last item pushed is the first item popped.

Queue

A first in, first out structure. The first item enqueued is the first item dequeued.

Linked list

A structure made from nodes. Each node stores data and a pointer to the next node.

Push

Add an item to the top of a stack.

Pop

Remove and return the item from the top of a stack.

Peek

Look at the next item to be removed without removing it.

Enqueue

Add an item to the rear of a queue.

Dequeue

Remove and return the item from the front of a queue.

Front pointer

A variable that identifies the front item in a queue.

Rear pointer

A variable that identifies the rear item in a queue.

Stack pointer

A variable that identifies the top position in a stack.

Pointer

A stored position or index that tells the algorithm where another item is located.

Start pointer

The pointer to the first node in a linked list.

Null pointer

A special value, often -1 in exam-style pseudocode, meaning no next node.

Free list

A list of unused node positions available for insertion.

Overflow

An error that happens when trying to add data to a full fixed-size structure.

Underflow

An error that happens when trying to remove data from an empty structure.

LIFO

Last in, first out. This describes a stack.

FIFO

First in, first out. This describes a queue.
